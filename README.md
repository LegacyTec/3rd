 ## :dart: Contextualização

Atualmente, enfrentamos desafios na gestão de talentos internos, especialmente na identificação 
de habilidades específicas entre os colaboradores. Com o crescimento das áreas, torna-se cada vez 
mais difícil saber, por exemplo, se há alguém com experiência em bancos de dados internamente.  
 
Propomos o desenvolvimento de uma plataforma semelhante a um "LinkedIn interno", onde cada 
colaborador possa criar e manter um perfil profissional. Nesse perfil, ele poderá registrar suas 
competências técnicas, soft skills, certificações, experiências anteriores e demais informações 
relevantes. Além disso, os gestores poderão complementar esses perfis com tags e avaliações, 
facilitando a identificação de talentos com base em critérios objetivos. Com essa ferramenta, será 
possível realizar buscas por habilidades específicas, avaliar o próprio departamento e etc.

## :page_facing_up: Requisitos Não Funcionais:  

> * Manual de Instalação (requisito Fatec – obrigatório, no Git); 
> * Manual do Usuário (requisito Fatec – obrigatório); 
> * Front Minimalista (requisito Fatec – obrigatório); 
> * Modelo de banco de dados relacional estruturado conforme o modelo entidade
relacionamento (MER), devidamente normalizado até a terceira forma normal (3FN), 
visando garantir integridade e eficiência no armazenamento dos dados. (requisito Fatec – 
obrigatório); 
> * A modelagem de dados deve ser documentada e validada antes do início da 
implementação de cada sprint, incluindo os modelos conceitual, lógico, físico e o 
dicionário de dados (Documentação detalhada que descreve cada elemento do modelo — 
como tabelas, campos, tipos de dados, restrições, chaves, descrições e relacionamentos.) 
(Requisito Fatec – obrigatório); 
> * Documentação API; 
> * Ter no mínimo 3 tipos de perfis com a seguinte hierarquia: diretor, supervisor (responde a 
um diretor) e colaborador (responde a um supervisor); 
> * Quantidade mínima de 100 registros, sendo no mínimo 7 diretores e mínimo de 2 
supervisores para cada diretor.
